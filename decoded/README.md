# Decoded Diamond Rush Data

This directory contains two different kinds of PNG output.

## `preview/`

`decoded/preview` is generated by `tools/drinspect`.

Those PNG files are diagnostic raw-ID color maps, not original art. Each color represents one raw byte ID from the decoded stage layers. They are useful for checking map coordinates, layer alignment, and tile/object frequency.

Useful files:

- `decoded/preview/index.html`: all decoded stage inspectors.
- `decoded/preview/world0/stage00-contactsheet.png`: raw-ID background/foreground/player/composite panels.
- `decoded/preview/id-index.md`: raw-ID frequency table.
- `decoded/preview/palette.png`: color legend for raw byte IDs `0..255`.

## `sprites/`

`decoded/sprites` is generated by `tools/drsprite`.

Those PNG files are extracted original Gameloft `.f` sprite sheets plus JSON frame/animation metadata. Use this directory when checking real pixel art.

Useful files:

- `decoded/sprites/index.html`: browser index for extracted sprite sheets.
- `decoded/sprites/manifest.json`: parse/export metadata.
- `decoded/sprites/0/chunk02-frames.png`: Angkor world tile/frame sheet.
- `decoded/sprites/o/chunk00-frames.png`: hero frame sheet.

Raw stage IDs do not directly equal sprite frame IDs. The Java renderer maps layer bytes and object state to sprite resources first, so full-stage visual reconstruction still needs that render mapping.
