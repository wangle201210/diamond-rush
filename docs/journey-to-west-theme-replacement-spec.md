# 西游主题高清视听与场景重构规范（第一阶段）

## 1. 目的

本文档用于在**完整保留当前关卡、路线、机关坐标、玩法状态机和通关逻辑**的前提下，把当前运行时替换为高清西游题材视听表现。第一阶段目标是：

1. 冻结现有世界地图连接、每关尺寸、三层数据、谜题依赖、敌人和机关位置。
2. 保留格子移动、推石、重力、钥匙、机关、锤击、钩取、检查点和Boss等全部玩法。
3. 全量替换品牌、角色、美术、动画、Logo、字体、音乐、音效和对话内容。
4. 把视觉母版提升为4倍分辨率，同时不改变逻辑尺寸、运动速度或碰撞结果。
5. 让每一个旧运行时视听资源槽都有明确的新资源承接或经源码证据确认的删除结论，避免遗漏后回退到原版素材。
6. 建立可审计的素材来源和验收流程。
7. 让生成素材服从真实关卡场景、机关功能和跨世界进程，不做脱离布局的逐图换名。

本文中的“1:1 替换”指**功能槽位一一覆盖且关卡行为保持不变**，不表示像素、轮廓、构图、帧拆分、配色或音乐旋律一一对应。关卡坐标和谜题路线在第一阶段明确保持不变。

每个原资源块、动画子槽、输出路径和生成brief的制作级结论见 [`journey-to-west-asset-replacement-catalog.md`](./journey-to-west-asset-replacement-catalog.md)。该目录是逐项替换时的权威清单；本文只维护总体边界。

《西游记》原著故事、古典人物和数百年前形成的基础情节可以作为公共领域题材使用，但现代影视、动画、游戏、插画、字体、配音和改编文本通常仍有各自的权利。项目只能依据原著概念和自行撰写的设计稿创作，不得借用现代作品的具体表达。

- 原著中的通用人物名称、地点、法器、妖怪类别和基础事件。
- 中国传统神话、建筑、纹样、服饰和器物的通用文化元素。
- 团队独立撰写的角色设定、对白、关卡故事和视觉规范。

- 根据本规范文字描述生成，并经过人工重绘和相似性检查的生成式素材。

禁止作为视觉或音频参考直接使用：

### 3.3 世界观、主角与叙事

#### 3.3.1 世界背景

人间山川之间存在由地脉、香火和行旅愿力凝成的`云路`。云路不是普通道路，而是连接洞天、城镇和古寺的灵性航道；每个大区域由一枚`经卷灵印`长期稳定，沿路的小型`云路印`则记录旅人已经获得的通行资格，并让云路驿站短暂接续尚未完全修复的路段。

故事开始时，花果山山门的五行封印失衡，三条主要云路相继断裂。花果山、火焰山和雪岭的守关妖王各自占据了一枚经卷灵印，区域内原本用于引水、导风、镇火和护寺的设施也因此失序。土地神无法离开辖地，只能借神龛和寻经针引导一名能够穿行洞天的年轻石猴修复路线。

主线目标是恢复三地云路并收回三枚经卷灵印。秘密关是旧云路坍塌后遗留的维护支路、封存库和镇妖廊，不是与主线无关的奖励异空间。世界解锁仍只由云路印的`10/25`累计门槛决定：云路印让驿站临时搭起可通行路径，经卷灵印则恢复该区域的长期稳定，因此玩家先跨界、后挑战Boss在叙事上同样成立。

#### 3.3.2 主角悟乐

- 主角正式名为`悟乐`，读音固定为“wù lè”，拉丁转写固定为`Wule`，资源slug固定为`wule`；中文界面不得只写“石猴行者”，英文界面不得拆写为`Wu Le`，也不得写成`Wukong`或`Monkey King`。
- 悟乐是一名年轻的石猴行者，不是孙悟空的改名版本，也不直接继承任何现代改编角色的身世、服装或台词。他从花果山一块吸收了多年香火的废弃云路碑中化生，因此能感知断裂路线；他因天性好奇、喜欢帮助行旅而接受土地神的托付。
- “悟”代表观察机关、理解因果，“乐”代表他面对险境仍轻快乐观。对白语气应简短、机敏、有温度，不使用傲慢帝王口吻，也不照搬古典或现代作品中的标志性台词。
- 悟乐以灵巧、观察和借力解谜为主：推石、引落、钩取、冻结和利用敌人压阵都应表现为因势利导，而不是依靠体型或蛮力碾压一切。
- 固定外观基线为深棕短毛、青绿色短褂、朱红腰绳和乌金护腕。建立悟乐的三视图、比例表、表情表、色板及法宝持握表后，才能制作游戏精灵。
- 不采用任何现代影视或游戏中的面部、盔甲、围巾、发冠、武器比例和姿态；宣传文案也不能暗示悟乐就是某个现代版本的孙悟空。

#### 3.3.5 叙事边界

- 导师固定为原创土地神形象，通过神龛、教程头像和少量对白出现；不能复制原作探险者头像构图。
- 悟乐可以按云路印门槛提前跨世界，因此对白只能引导推荐顺序，不能宣称击败前一Boss才有资格进入下一地。
- 世界观只解释现有状态，不新增伙伴系统、法力值、任务物品槽、第四种货币或额外结局条件。
- 每个机关必须在场景中有材质来源、动力来源和维护关系。例如风瓮连接风道、镜柱属于古寺寒阵、地火口接入岩缝；禁止把单件素材孤立贴进不相关房间。
- 主线对白统一围绕悟乐、云路、土地神、三枚经卷灵印和三地失序展开，不沿用原游戏的探险对白和徽记叙事。

### 3.4 叙事、经济与工具进程

第一阶段虽然冻结关卡数据，但新叙事必须解释现有经济和回访关系，不能只给旧对象改名：

- `灵蕴珠`是散落在关卡中的普通灵气。它同时承担关内功德封印配额、结算收集率和云路行囊坊货币；商店按`150/400/1000/3000`实际扣除灵蕴珠，逐级把最大护体毫光从`5`提升到`8`。
- `云路印`是朱砂小印，不是药品。它是永久累计的路线资质，达到`10`枚可通往火焰山，达到`25`枚可通往雪岭；源码只检查累计门槛，不扣除数量，因此文案不得使用“购买世界”或“消耗云路印”。
- `经卷灵印`是每个世界Boss守护的大型主线信物，与可累计的云路印必须在名称、轮廓、尺寸和HUD位置上明显区分。
- 云路显现与Boss灵印是两条独立进度。源码进入火焰山/雪岭只检查云路印累计值，不检查前一世界的灵印位；地图、对白和存档迁移不得把“击败Boss”追加成下一世界的硬门槛。
- 原文本中的`Hidden Crystal`/`Fire Crystal`不建立第四套收集进度，统一重写为对应世界的`经卷灵印`。源码持久化只有三枚世界灵印位、灵蕴珠银行和云路印累计等相关状态；不得为换主题凭空增加“每世界一页”的计数、HUD槽或结局条件。
- 花果山Stage 4的raw `24`经匣是`如意铁棒`的标准取得点；火焰山Stage 3的raw `27`经匣是`幌金绳`的标准取得点；雪岭Stage 6的raw `26`经匣把已有铁棒淬炼成`玄冰铁棒`，随后才开启冻结敌人和跨世界隐藏路线。
- 为兼容云路印允许的提前跨世界顺序，源码进入火焰山时会把缺失工具补到等级`1`，进入雪岭时补到等级`2`。新叙事可用界门上的静态法宝纹样或非阻塞文案解释为土地神补发基础法宝，但不得增加确认框/演出时长、伪造第二个经匣、播放不存在的开箱演出，或把这个兜底错误提升为玄冰铁棒。
- `玄冰铁棒`是同一工具的强化形态，不是第四件并列武器。工具栏、举起动画和对白都应表现“淬炼/觉醒”，而不是凭空获得另一根铁棒。
- 三个Boss分别守护所在世界的经卷灵印：碧鳞蟒王占据花果山祭坛，铁甲牛将控制火焰山铸坊与风水机关，寒魄狮王维持雪岭古寺寒阵。Boss场地、机关材质和奖励演出必须支持这一关系。

跨世界流程必须在地图、关卡标题、奖励对白和场景美术中保持同一因果链：

```text
教程取得寻经针
  -> 花果山Stage 4标准取得如意铁棒；若提前跨界，火焰山界门仅补足该基础能力
  -> 云路印达到10，火焰山云路显现（不要求已取得花果山灵印）
  -> 火焰山Stage 3标准取得幌金绳；若继续提前跨界，雪岭界门仅补足该牵引能力
  -> 云路印达到25，雪岭云路显现（不要求已取得火焰山灵印）
  -> 雪岭淬炼玄冰铁棒，并回访各世界冻结隐藏路线
  -> 分别击败碧鳞蟒王、铁甲牛将和寒魄狮王，收齐三枚经卷灵印
```

上图描述能力和路线的依赖，不规定Boss挑战顺序。叙事可以引导玩家先完成当前世界，但不能通过美术提示、按钮状态或新增存档条件改变源码允许的顺序。

## 4. 技术兼容规则

### 4.1 不变的运行时约束

- 逻辑画布保持 `240x320`；不得把高清像素写回逻辑坐标。
- 逻辑上HUD `40px`、游戏区 `240px`、下HUD `40px`。
- 逻辑格保持 `24x24px`。
- 权威玩法状态保持 `20 TPS`；是否采用60Hz平滑显示属于独立渲染层问题。
- 碰撞、拾取、伤害和机关判定继续由逻辑数据控制，不能从新图片的非透明区域推导。

### 4.4 高清渲染合同

高清视听与场景重构采用固定4倍视觉比例：

| 项目 | 逻辑尺寸 | 高清视觉尺寸 |
| --- | ---: | ---: |
| 完整画布 | `240x320` | `960x1280` |
| 单格 | `24x24` | `96x96` |
| 顶部HUD | `240x40` | `960x160` |
| 游戏视口 | `240x240` | `960x960` |
| 底部HUD | `240x40` | `960x160` |

实现规则：

- `renderScale=4` 只存在于绘制层；Runtime继续使用逻辑坐标。
- 摄像机、对象offset、绳索长度和粒子发射点先按逻辑值计算，再统一乘以4。
- 新PNG直接按4倍尺寸制作，不能先在24px格中画完再用插值放大。
- pivot同时记录逻辑值和高清像素值；高清值必须等于逻辑值乘4，除非是纯视觉越界。
- 角色脚底、石头中心、门轴、钉柱接缝和Boss占用格必须与逻辑格严格对齐。
- 允许角色头饰、武器、火焰和粒子超出96px格，但要声明最大visual bounds，避免相机边缘裁切。
- HUD、Logo、头像和字体使用原生高清资源，不从低清图放大。
- 动画可增加视觉中间帧；命中、发奖、开门、伤害和状态切换仍发生在原20Hz逻辑tick。
- 运行时按当前世界和关卡懒加载高清atlas，避免把121个资源块的4倍纹理一次性常驻显存。
- 每个atlas建议不超过GPU目标平台的安全纹理上限，并记录加载后纹理内存；不能用巨型透明画布换取排版方便。
- 第一阶段默认采用“高清国风彩绘2D”：清晰轮廓、适度传统纹样、受控材质细节和统一光向。不能把低清像素角色与高清写实地砖混用。

## 12. 生成式素材规则

GPT-image等工具可以参与创作，但应按以下流程使用：

1. 先从真实关卡数据建立scene ID、几何遮罩、机制清单和文字brief，不附原版或现代西游作品截图。
2. 每次生成必须引用制作目录第2.6至2.8节的scene ID，并声明功能职责、占格、状态、相邻材质和输出模式。
3. 先生成角色、物体或多格构件的原创设计母版，再以团队自己的母版作为连续性参考。
4. 不直接要求模型输出最终精灵表；动画帧由人工按pivot、动作弧线和逻辑事件tick整理。
5. 多格构件先生成完整构图再切片；场景板只做材质和层级验证，不能替代关卡JSON渲染。
6. 对高清精灵进行人工清边、统一光向、轮廓修正、比例校正和逐帧抖动检查。
7. 保存原始生成图、提示词、scene ID、日期、模型名称、编辑者和最终导出文件。
8. 对Logo、主角、Boss和关键UI执行人工相似性审查；发现明显联想到特定现代作品时退回重做。
9. 不使用不明来源的LoRA、模型、字体、音色库或样本包。

建议提示词只描述：

- 公共领域题材和原创角色设定。
- 画布、视角、动作、输出模式和色数要求；只有精灵使用透明背景，地砖和场景板按各自模式输出。
- 自己定义的轮廓、材质、光向和配色。

提示词中不应出现需要模仿的现代作品、公司、艺术家或原游戏名称。

## 15. 推荐迁移顺序

1. 冻结当前原版研究运行时，仅作为私有行为参考。
2. 给全部现有关卡生成状态哈希和正式路线基线，保证素材任务不改玩法。

11. 对发行包执行视听独立性扫描及人工相似性审查。
